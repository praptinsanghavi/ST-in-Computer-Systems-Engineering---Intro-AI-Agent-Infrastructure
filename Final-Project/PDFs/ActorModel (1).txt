Actor Terms
Actor = a class that can contain state and modifies
that state by processing messages it receives.
 ActorReference = a handle to an actor. Allows you
to send the actor messages without knowing its
implementation type or location on network.
 ActorSystem = a collection of actors that exist
inside a single process and communicate via inmemory message passing.
 Cluster = a collection of networked ActorSystems
whose actors communicate via TCP message
passing.

Operations
43



The only allowed operations for an actor are:
 to create another actor,
 to send a message
 or to designate how to handle the next message

First two operations are quite easy to understand.
 Let’s focus on the last one.


An Actor can hold its own private state.
It can decide how to process the next message based on that state.

44

Imagine an Actor storing a total balance of our account.

45

If an Actor receives a message with a new transaction, it updates
its state by adding the new amount to the already calculated total
balance.
This means, for the next message the state of the Actor will be
different.

Actors in More Details
48

Actors are Isolated.
They do not share memory

way to change it is by

Every Actor has its own
mailbox that is similar to a
Message Queue.

Messages
49

Messages are stored in Actor’s Mailboxes until they are processed.
Actors, after created, wait for messages to arrive.
Actors can communicate with each other only through messages.
Messages are sent to Actors’ mailboxes, and processed in FIFO order.
Messages are simple immutable data structures that are easy to send over the network.
Actors can process one message at a time.

Actors are decoupled
58

Actors work asynchronously.
 They don’t need to wait for a response form another Actor.
 Actors have addresses.
 It is possible for an Actor to send a message to another Actor by knowing its
Address.


• Actors have addresses, so it’s possible for an actor to send a message to another
actor by knowing its address.
• An actor can only communicate with actors whose addresses it has.
• An actor has addresses of the actors it has itself created and it can obtain other
addresses from a message it receives.
• One actor can have many addresses.
• We need to remember that address is not equal to identity.
