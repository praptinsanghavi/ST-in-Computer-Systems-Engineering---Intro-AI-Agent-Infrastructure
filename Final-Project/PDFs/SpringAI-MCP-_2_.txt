´é¿    Even though we typically refer to tool calling as a model capability, it is
     actually up to the client application to provide the tool calling logic.

´é¿    The model can only request a tool call and provide the input arguments,
     whereas the application is responsible for executing the tool call from the
     input arguments and returning the result.

´é¿    The model never gets access to any of the APIs provided as tools, which is
     a critical security consideration.

´é¿    Spring AI provides convenient APIs to define tools, resolve tool call requests
     from a model, and execute the tool calls.

Model Context Protocol (MCP)
22


´é¿    The Model Context Protocol (MCP) is a standardized protocol that enables
     AI models to interact with external tools and resources in a structured way.
´é¿    It supports multiple transport mechanisms to provide flexibility across
     different environments.
´é¿    The MCP Java SDK provides a Java implementation of the Model Context
     Protocol, enabling standardized interaction with AI models and tools through
     both synchronous and asynchronous communication patterns.
´é¿    Spring AI MCP extends the MCP Java SDK with Spring Boot integration,
     providing both client and server starters.
´é¿    Bootstrap your AI applications with MCP support using Spring Initializer.
MCP
23


´é¿    Modern web applications are increasingly integrating with Large Language
     Models (LLMs) to build solutions, which are not just limited to general
     knowledge-based question-answering.
´é¿    To enhance an AI modelÔÇÖs response and make it more context-aware, we
     can connect it to external sources like search engines, databases, and file
     systems.
´é¿    However, integrating and managing multiple data sources with different
     formats and protocols is a challenge.
´é¿    The Model Context Protocol (MCP), introduced by Anthropic, addresses this
     integration challenge and provides a standardized way to connect AI-
     powered applications with external data sources.
´é¿    Through MCP, we can build complex agents and workflows on top of a
     native LLM.

Communication between clients and servers
25


´é¿    Additionally, to handle communication between clients and servers, MCP
     provides two transport channels.

´é¿    To enable communication through standard input and output streams with
     local processes and command-line tools, it provides the Standard
     Input/Output (stdio) transport type.

´é¿    Alternatively, for HTTP-based communication between clients and servers,
     it provides the Server-Sent Events (SSE) transport type.

Model Context Protocol (MCP) Client
28

´é¿    The MCP Client is a key component in the MCP architecture, responsible for
     establishing and managing connections with MCP servers.
´é¿    It implements the client-side of the protocol, handling:
      ´éñ Protocol version negotiation to ensure compatibility with servers
      ´éñ Capability negotiation to determine available features
      ´éñ Message transport and JSON-RPC communication
      ´éñ Tool discovery and execution
      ´éñ Resource access and management
      ´éñ Prompt system interactions
      ´éñ Optional features:
          ´ü« Roots management
          ´ü« Sampling support
      ´éñ Synchronous and asynchronous operations
      ´éñ Transport options:
          ´ü« Stdio-based transport for process-based communication
          ´ü« Java HttpClient-based SSE client transport
          ´ü« WebFlux SSE client transport for reactive HTTP streaming

´é¿    The MCP Server is a foundational component in the Model Context Protocol
     (MCP) architecture that provides tools, resources, and capabilities to clients.
´é¿    It implements the server-side of the protocol, responsible for:
      ´éñ Server-side protocol operations implementation
          ´ü« Tool exposure and discovery
          ´ü« Resource management with URI-based access
          ´ü« Prompt template provision and handling
          ´ü« Capability negotiation with clients
          ´ü« Structured logging and notifications
      ´éñ Concurrent client connection management
      ´éñ Synchronous and Asynchronous API support
      ´éñ Transport implementations:
          ´ü« Stdio-based transport for process-based communication
          ´ü« Servlet-based SSE server transport
          ´ü« WebFlux SSE server transport for reactive HTTP streaming
          ´ü« WebMVC SSE server transport for servlet-based HTTP streaming

´é¿    Prompts are the inputs that guide an AI model to generate specific outputs.
     The design and phrasing of these prompts significantly influence the
     modelÔÇÖs responses.
´é¿    The structure of prompts has evolved over time within the AI field.
´é¿    Initially, prompts were simple strings.
´é¿    Over time, they grew to include placeholders for specific inputs, like
     "USER:", which the AI model recognizes.
´é¿    OpenAI have introduced even more structure to prompts by categorizing
     multiple message strings into distinct roles before they are processed by the
     AI model.
