´é¿

Give models access to new functionality and data they can use to follow

´é¿

Function calling (also known as tool calling) provides a powerful and flexible way
for OpenAI models to interface with external systems and access data outside their
training data.

´é¿

Tool calling (or function calling) enables LLMs to interact with external systems,
APIs, and databases, allowing AI to perform actions rather than just generating text

´é¿

The model intelligently analyzes user queries, determines if a tool (defined via
JSON schema) is necessary, and outputs structured arguments for execution.

When we make an API request to the model with a prompt, we can include

2. Tool calls - requests from the model to use tools
6

´é¿

A function call or tool call refers to a special kind of response we can get
from the model if it examines a prompt, and then determines that in order to
follow the instructions in the prompt, it needs to call one of the tools we
made available to it.

´é¿

If the model receives a prompt like ÔÇ£what is the weather in Paris?ÔÇØ
´éñ in an API request, it could respond to that prompt with a tool call for the
get_weather tool, with Paris as the location argument.

3. Tool call outputs - output we generate for the model
7

A function call output or tool call output refers to the response a tool generates
using the input from a modelÔÇÖs tool call.
´é¿
The tool call output can either be structured JSON or plain text, and it should
contain a reference to a specific model tool call (referenced by call_id in the
examples to come).
´é¿
To complete our weather example:
´é¿
The model has access to a get_weather tool that takes location as an argument.

´é¿

"C"}, indicating a current temperature of 25 degrees).
´é¿
We then send all of the tool definition, the original prompt, the modelÔÇÖs tool call, and
the tool call output back to the model to finally receive a text response like:
´éñ The weather in Paris today is 25C.

Functions versus tools
8

´é¿

A function is a specific kind of tool, defined by a JSON schema.

´é¿

A function definition allows the model to pass data to your application,
where your code can access data or take actions suggested by the model.

´é¿

In addition to function tools, there are custom tools that work with free text
inputs and outputs.

´é¿

There are also built-in tools that are part of the OpenAI platform.

´é¿

These tools enable the model to search the web, execute code, access the
functionality of an MCP server, and more.

The tool calling flow
10

Tool calling is a multi-step conversation between your application and a
model via the OpenAI API.
´é¿ The tool calling flow has five high level steps:
1.
Make a request to the model with tools it could call
2.
Receive a tool call from the model
3.
Execute code on the application side with input from the tool call
4.
Make a second request to the model with the tool output
5.
Receive a final response from the model (or more tool calls)
