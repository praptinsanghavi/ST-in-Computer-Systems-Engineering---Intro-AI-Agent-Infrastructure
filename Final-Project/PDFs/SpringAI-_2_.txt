1. Chat Model API
3


´é¿   The Chat Model API offers developers the ability to integrate AI-powered chat
    completion capabilities into their applications.


´é¿   It leverages pre-trained language models, such as GPT, to generate human-like
    responses to user inputs in natural language.


´é¿   The API typically works by sending a prompt or partial conversation to the AI
    model, which then generates a completion or continuation of the conversation
    based on its training data and understanding of natural language patterns.


´é¿   The completed response is then returned to the application, which can present
    it to the user or use it for further processing.
The Spring AI Chat Model API
4


´é¿   The Spring AI Chat Model API is designed to be a simple and portable interface
    for interacting with various AI Models, allowing developers to switch between
    different models with minimal code changes.


´é¿   This design aligns with SpringÔÇÖs philosophy of modularity and
    interchangeability.


´é¿   Also, with the help of companion classes like Prompt for input encapsulation
    and ChatResponse for output handling, the Chat Model API unifies the
    communication with AI Models.

´é¿   It manages the complexity of request preparation and response parsing,
    offering a direct and simplified API interaction.

Primary Goals
24


´é¿    The design of the EmbeddingModel interface centers around two primary goals:
     ´éñ Portability:
        ´ü« This interface ensures easy adaptability across various embedding models.
        ´ü« It allows developers to switch between different embedding techniques or
          models with minimal code changes.
        ´ü« This design aligns with SpringÔÇÖs philosophy of modularity and
          interchangeability.
     ´éñ Simplicity:
        ´ü« EmbeddingModel simplifies the process of converting text to embeddings.
        ´ü« By providing straightforward methods like embed(String text) and
          embed(Document document), it takes the complexity out of dealing with raw
          text data and embedding algorithms.
        ´ü« This design choice makes it easier for developers, especially those new to AI,
          to utilize embeddings in their applications without delving deep into the
          underlying mechanics.
API Overview
25


´é¿    The Embedding Model API is built on top of the generic Spring AI Model
     API, which is a part of the Spring AI library.

´é¿    As such, the EmbeddingModel interface extends the Model interface, which
     provides a standard set of methods for interacting with AI models.

´é¿    The EmbeddingRequest and EmbeddingResponse classes extend from the
     ModelRequest and ModelResponse are used to encapsulate the input and
     output of the embedding models, respectively.

´é¿    The Embedding API in turn is used by higher-level components to
     implement Embedding Models for specific embedding models, such as
     OpenAI, Titan, Azure OpenAI, Ollie, and others.

Azure OpenAI Image Generation
30


´é¿    Spring AI supports DALL-E, the Image generation model from Azure OpenAI.
´é¿    Obtain your Azure OpenAI endpoint and api-key from the Azure OpenAI Service
     section on the Azure Portal.
´é¿    Spring AI defines a configuration property named spring.ai.azure.openai.api-key
     that you should set to the value of the API Key obtained from Azure.
´é¿    There is also a configuration property named spring.ai.azure.openai.endpoint that
     you should set to the endpoint URL obtained when provisioning your model in
     Azure.
´é¿    Exporting environment variables is one way to set these configuration properties:
´é¿    export SPRING_AI_AZURE_OPENAI_API_KEY=<INSERT KEY HERE>
´é¿    export SPRING_AI_AZURE_OPENAI_ENDPOINT=<INSERT ENDPOINT URL HERE>
